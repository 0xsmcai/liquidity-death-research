# Post-Launch Liquidity Death on Token Launchpads

## The Problem

Tokens graduate from bonding curves into DEX pools, and within 48 hours the pool becomes a graveyard — locked liquidity earning nothing, no active LPs stepping in, volume craters, price spirals down. Creator fees flow to passive wallets instead of being put back to work. The capital is there but nobody's managing it.

## The Hypothesis

An autonomous agent sitting at the creator fee address that actively recycles fees into managed LP positions — turning a passive tax into a liquidity engine.

## Status

**Research phase.** No proven model yet. Investigating whether this is viable.

## Key Questions

- Can creator fees generate enough yield to sustain active LP management after gas and IL?
- What's the minimum volume threshold for the flywheel to be self-sustaining?
- What rebalancing strategy minimizes IL while maintaining tight spreads?
- Can this be standardized into a reusable template for any launchpad?
- What on-chain transparency is needed for trust?
